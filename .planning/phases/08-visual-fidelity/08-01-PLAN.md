<objective>
Extend the seat domain to represent the full cloud_2 lifecycle by adding two new statuses — `checked_in` (passenger has checked in but not boarded) and `boarded` (passenger has physically boarded) — and propagate them through the API mapper, presentation classes, and stylesheet.

Purpose: cloud_2 visually distinguishes O / CKIN / BDED with three separate colors. The current poc_dcs_fe domain collapses all three into `occupied`, which is the single largest contributor to the ~55% visual fidelity gap. Closing this gap is a prerequisite for plans 08-02 and 08-03.

Output: An exhaustive 9-state SeatStatus union, an updated mapper that consumes API codes `C` and `D`, presentation classes `seat_checked_in` / `seat_boarded`, and matching CSS rules using cloud_2 token colors.
</objective>

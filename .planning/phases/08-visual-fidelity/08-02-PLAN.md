<objective>
Add a sticky column-header row (A, B, C, D, E, F …) above the cabin deck's seat rows, mirroring cloud_2's `seat_plan_column_labels` element. This provides the visual anchor that lets passengers correlate a seat block with its column letter when scrolling long-haul layouts.

Purpose: Closes one of the most visible audit gaps — cloud_2 always shows column headers; poc_dcs_fe currently shows none. Without this users cannot tell column position at a glance on dense decks.

Output: A new `SeatColumnHeaders` component with tests, a CabinDeck integration that derives column letters from the first row, and CSS that sticks the header to the scroll container's top.
</objective>

1. Create `src/presentation/components/SeatColumnHeaders.tsx`:
   ```tsx
   interface SeatColumnHeadersProps {
     columns: string[];
   }

   export function SeatColumnHeaders({ columns }: SeatColumnHeadersProps) {
     if (columns.length === 0) return null;
     return (
       <div className="seat_column_headers" role="row" aria-label="Seat columns">
         {columns.map((letter) => (
           <span key={letter} className="column_label" role="columnheader">
             {letter}
           </span>
         ))}
       </div>
     );
   }
   ```
   Decide the empty-state contract: return `null` (matches Test 3 expectation that `container.firstChild` is null). Add `role="row"` + `role="columnheader"` for accessibility — cloud_2's plain-div version has no a11y; we improve here at zero cost.

2. Export from `src/presentation/components/index.ts` (add the line; preserve alphabetical order if the file is already sorted).

3. Update `CabinDeck.tsx`:
   ```tsx
   import { SeatColumnHeaders } from './SeatColumnHeaders';

   export function CabinDeck({ rows, selectedSeat, onSeatSelect }: CabinDeckProps) {
     const columns = rows[0]?.seats.map((seat) => seat.number.value.slice(-1)) ?? [];
     return (
       <div className="seat_map_container">
         <SeatColumnHeaders columns={columns} />
         {rows.map((row) => (
           <SeatRow key={row.rowNumber} row={row} selectedSeat={selectedSeat} onSeatSelect={onSeatSelect} />
         ))}
       </div>
     );
   }
   ```
   Derive column letters from `rows[0].seats`. The seat number ends with the column letter (e.g., `12A` → `A`). If `rows` is empty, `columns` becomes `[]` and `SeatColumnHeaders` renders null — safe for empty-deck states.

   Note: verify the actual property path is `seat.number.value` by reading `src/domain/seat/SeatNumber.ts` first. If the public accessor differs (e.g., `seat.number.toString()`), adapt accordingly. Do NOT assume — read first.

4. Add a test in `CabinDeck.test.tsx` (create the file if it does not exist, otherwise append):
   `it('renders column headers derived from row 0', () => ...)` — pass a `rows` prop with one row containing seats `12A`, `12B`, `12C`, render, assert `.seat_column_headers` contains exactly `['A','B','C']`.

WHY: Deriving from `rows[0]` matches cloud_2's behavior — the header reflects the densest column layout, which is universally row 0 in the cabin domain. Centralizing this in CabinDeck (not SeatRow) lets the header be sticky over the scroll container; SeatRow scrolls with its content.

AVOID:
- Do NOT compute the union of columns across all rows (cloud_2 does not; mismatched-column rows are an edge case handled by per-cell aisle gaps in SeatRow).
- Do NOT add column-header rendering to SeatRow — sticky positioning needs a single header sibling, not one per row.
- Do NOT hardcode `['A','B','C','D','E','F']` — derive from data.
- Do NOT modify SeatRow in this plan (out of scope).

  <action>
Add stylesheet rules that make `.seat_column_headers` visually align with the seat grid below it and stick to the top of its scroll container.

Add to `src/styles/layout.css`:
```css
.seat_column_headers {
  position: sticky;
  top: 0;
  z-index: 10;
  display: flex;
  gap: var(--seat-gap);
  padding-left: 1.5rem;       /* offset to align with seat columns; matches the row-number column width */
  padding-bottom: var(--space-xs);
  background: var(--color-bg);  /* opaque so seats don't bleed through when scrolling */
  border-bottom: 1px solid var(--color-border);
  font-weight: var(--font-weight-bold);
  font-size: var(--font-size-xs);
  color: var(--color-seat-text-dark);
}

.column_label {
  width: var(--seat-size);
  text-align: center;
  flex-shrink: 0;
}
```

Notes:
- `padding-left: 1.5rem` matches the row-number column rendered by SeatRow. If SeatRow uses a different value (read it; do not assume), update this rule to match exactly. If the row-number column has no fixed width yet, add a `--row-number-width` token and reference it here AND in SeatRow in a follow-up — but do not change SeatRow in this plan.
- `position: sticky` requires a non-`visible` `overflow` on a parent. Verify `.seat_map_container` (or its parent) has `overflow: auto` or `overflow-y: auto`. If not, sticky becomes equivalent to relative — header will scroll away. Add `overflow-y: auto; max-height: ...` to `.seat_map_container` ONLY IF it does not already exist; do NOT change existing overflow values without explicit reason.
- `--color-bg` and `--color-border` should already be defined in `tokens.css` (verify with grep). If missing, fall back to `#fff` / `#e0e0e0` and note the missing tokens in SUMMARY.

WHY: cloud_2's column labels stay visible while the seat map scrolls, which is essential for wide-body planes (40+ rows). Without `position: sticky` the header scrolls away and the feature has no value.

AVOID:
- Do not use `position: fixed` — it ignores the scroll container and would sit over unrelated UI.
- Do not set `width: 100%` on the header — flex with gap handles sizing; explicit width fights the layout.
- Do not introduce JavaScript scroll-handlers; CSS sticky is sufficient for this use case.
- Do not change `--seat-size` or `--seat-gap` values (Plan 08-03 owns token edits).

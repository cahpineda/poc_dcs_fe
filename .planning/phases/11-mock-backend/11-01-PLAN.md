<objective>
Build a stateful Node.js/Express mock backend server that serves all seat_plan endpoints the React POC depends on, using in-memory data that mutates on each command call. This server replaces the real cloud_2 backend for local development and E2E testing.

Purpose: All subsequent phases (12-15) require real HTTP round-trips to validate features. The mock server provides deterministic, controllable data without needing a cloud_2 environment. It also documents the exact wire format each endpoint expects and returns.

Output:
- `server/` directory with standalone Node.js/Express app (plain JS, no TypeScript)
- `server/data/flights.js` — in-memory flight state for FL001 and FL002
- `server/routes/seatPlan.js` — GET endpoints for seat plan queries
- `server/routes/commands.js` — POST endpoints for all mutations
- `server/index.js` — Express app entry point with CORS
- `npm run server` script in root package.json
</objective>

<feature name="Command (mutation) endpoints">

<task type="auto">
  <name>Task 4: Build POST mutation endpoints</name>
  <complexity>medium</complexity>
  <files>server/routes/commands.js</files>
  <action>
Create `server/routes/commands.js` with all mutation endpoints. Each endpoint reads the body, validates required fields, mutates in-memory state, returns success.

**POST /ajax/seat_plan/assign_seat**
Body: `{ flight_id, seat_number, passenger_id, passenger_name? }`
Logic:
- If seat status is not 'A' (available) or 'EA' (exit_row_available) → return 409 `{ success:false, error:"Seat is already occupied" }`
- Otherwise: update seat to status='O', set passenger_key=passenger_id
→ return `{ success: true, data: { seat_number, new_status: "O" } }`

**POST /ajax/seat_plan/block_seat**
Body: `{ flight_id, seat_number, reason? }`
Logic: update seat status to 'B', set block_note=reason
→ return `{ success: true, data: { seat_number, new_status: "B" } }`

**POST /ajax/seat_plan/unblock_seat**
Body: `{ flight_id, seat_number }`
Logic: update seat status to 'A', clear block_note
→ return `{ success: true, data: { seat_number, new_status: "A" } }`

**POST /ajax/seat_plan/reseat_passenger**
Body: `{ flight_id, from_seat, to_seat, passenger_id }`
Validation: to_seat must be status 'A' or 'EA'
Logic:
- Copy passenger data from from_seat to to_seat (status='O')
- Reset from_seat to status='A', clear passenger fields
→ return `{ success: true, data: { from_seat, to_seat } }`

**POST /dc/unseat_passenger**
Body: `{ flight_id, seat_number }`
Logic: reset seat to status='A', clear all passenger fields
→ return `{ success: true, data: { seat_number, new_status: "A" } }`

**POST /dc/swap_seats**
Body: `{ flight_id, seat_a, seat_b }`
Logic: swap all passenger data between the two seats
→ return `{ success: true, data: { seat_a, seat_b } }`

**POST /dc/reseat_group**
Body: `{ flight_id, passenger_ids[], target_row? }`
Logic: find available seats in target_row (or next available row), move each passenger
→ return `{ success: true, data: { moved: [{from, to}] } }`
  </action>

  <done>All 7 mutation endpoints work and state persists within the server session</done>

Create `server/index.js`:

```js
const express = require('express');
const cors = require('cors');
const seatPlanRoutes = require('./routes/seatPlan');
const commandRoutes = require('./routes/commands');

const app = express();
const PORT = process.env.PORT || 3001;

app.use(cors({
  origin: ['http://localhost:5174', 'http://localhost:5173'],
  credentials: true,
}));
app.use(express.json());

// Health check
app.get('/health', (req, res) => res.json({ status: 'ok', flights: ['FL001', 'FL002'] }));

// Seat plan query routes
app.use('/ws/v1.8', seatPlanRoutes);

// Command routes
app.use('/ajax/seat_plan', commandRoutes);
app.use('/dc', commandRoutes);

app.listen(PORT, () => {
  console.log(`Mock DCS server running on http://localhost:${PORT}`);
  console.log(`Serving flights: FL001, FL002`);
});
```

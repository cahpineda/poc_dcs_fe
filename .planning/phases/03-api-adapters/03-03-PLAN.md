  <files>

  <behavior>
    Happy path: mapSeatPlanDTO(validCloud2Response) → SeatPlanResult with rows array, flightId, lower deck seats populated
    Null/invalid: mapSeatPlanDTO(null) → throws Error("Invalid seat plan response: null")
    Error/boundary: mapSeatPlanDTO with missing seats array → returns SeatPlanResult with empty rows (graceful degradation, not a throw)
    Additional: SeatPlanResult.totalSeats → sum of seats across all rows
    Additional: SeatPlanResult.availableCount → count of seats with isAvailable() === true
  </behavior>
  <implementation>
    Create `src/domain/seat/CabinRow.ts`:
    ```ts
    export interface CabinRow {
      readonly rowNumber: number;
      readonly seats: ReadonlyArray<Seat>;
      readonly isExitRow: boolean;
    }
    ```

    Create `src/domain/seat/SeatPlanResult.ts` as a class with:
    - Private constructor(public readonly rows: CabinRow[], public readonly flightId: string, public readonly isUpperDeck: boolean)
    - Static create(props): SeatPlanResult
    - get totalSeats(): number — sum of all seats
    - get availableCount(): number — count of available seats
    - getSeat(seatNumber: string): Seat | undefined

    Create `src/infrastructure/adapters/mappers/seatPlanMapper.ts`:
    ```ts
    export function mapSeatPlanDTO(dto: unknown): SeatPlanResult
    ```
    The mapper should:
    - Validate dto is not null/undefined
    - Map dto.flight_id or dto.flightId to flightId
    - Map dto.seat_rows[] → CabinRow[] → Seat[] using SeatNumber and Seat factories
    - Map cloud_2 seat status strings ('A'=available, 'O'=occupied, 'B'=blocked, etc.) to SeatStatus values
    - Return gracefully with empty rows if dto.seat_rows is missing

    Cloud_2 status mapping from MIGRATION-CONTEXT.md:
    - 'A' → 'available'
    - 'O' → 'occupied'
    - 'B' → 'blocked'
    - 'EA' → 'exit_row_available'
    - 'EO' → 'exit_row_occupied'
    - 'U' → 'unavailable'
    - 'I' → 'infant_occupied'

    For tests: use a fixture object matching cloud_2 ws_v1.8 seat_rows shape from MIGRATION-CONTEXT.md.
  </implementation>
</feature>

<objective>
Add visual indicators on seat cells for two safety/operational signals: infant on lap (small bottom-right dot) and seat-block alert (small bottom-left dot, hover shows block reason). Match cloud_2's `.seat_inf_alert_graphic` and `.seat_block_alert_graphic` overlays. Strict TDD — start with component-level tests since the visual overlay is the user-observable contract.
</objective>

Plan 09-01 established the pattern: Seat domain entity carries optional passenger fields, mapper threads them, SeatCell renders them. This plan applies the same pattern to two more fields:

- `has_infant` (boolean) — passenger has a lap infant. Render a small visual indicator (.seat_infant_indicator) at bottom-right of the seat cell.
- `block_note` (string | undefined) — staff note explaining why a seat is blocked. Render a small alert dot (.seat_block_indicator) at bottom-left, with the note as a `title` attribute for hover tooltip.

Both are stacked overlays that don't replace any existing seat content. They are pure presentation; the button stays clickable for available statuses.

Cloud_2 sizes: infant icon 11x15px, block icon 15x15px. POC uses simplified colored dots tied to design tokens (`--color-seat-infant`, `--color-seat-blocked`) since the SVG glyphs are out of scope for the POC.

Position: `.seat_cell` will become `position: relative` (it may already be — verify in seat-states.css) so the absolute-positioned indicators anchor correctly.
</context>

<objective>
Add passenger context (initials) overlay to occupied/checked_in/boarded seats so agent users can see who is in each seat at a glance, matching cloud_2 behavior. Extend Seat domain entity with optional passengerName, expose computed passengerInitials, thread through mapper and presentation layer using strict TDD.
</objective>

Phase 8 added checked_in/boarded statuses and column headers. Phase 9 layers passenger context overlays directly onto seat cells. This plan addresses the most visible overlay: a 2-character initials label that replaces the seat number for any seat that has a passenger assigned.

Cloud_2 reference: agent UI shows `<div class="seat_cell seat_occupied">JD</div>` instead of `<div class="seat_cell seat_occupied">12A</div>` for seat 12A occupied by JOHN DOE. Initials algorithm: first character of name + last character of name (handles single-word names without crashing).

Current Seat entity is immutable, constructed via Seat.create. No passenger fields exist yet. SeatCell currently renders `{seatNumber}` literally; we will swap to `{passengerInitials ?? seatNumber}` for occupied-family statuses only.
</context>

## Task 1.3 — Implement Seat.passengerName + passengerInitials

**files:**
- src/domain/seat/Seat.ts

**action:** Make the failing tests in Task 1.1 pass with minimum production code. Specifically:

1. Extend `interface SeatProps`:
   ```ts
   interface SeatProps {
     seatNumber: SeatNumber;
     status: SeatStatus;
     cabinClass: string;
     passengerName?: string;
   }
   ```

2. Add a private readonly field `_passengerName: string | null` to the constructor (normalize empty/whitespace to null at construction time).

3. Update `Seat.create`:
   ```ts
   static create(props: SeatProps): Seat {
     if (!props.seatNumber) throw new Error('Seat requires a SeatNumber');
     const trimmed = props.passengerName?.trim();
     const passengerName = trimmed && trimmed.length > 0 ? trimmed : null;
     return new Seat(props.seatNumber, props.status, props.cabinClass, passengerName);
   }
   ```

4. Add the getter:
   ```ts
   get passengerInitials(): string | null {
     if (!this._passengerName) return null;
     const name = this._passengerName.toUpperCase();
     const first = name.charAt(0);
     const last = name.charAt(name.length - 1);
     return `${first}${last}`;
   }
   ```

5. Update the existing `withStatus` method to preserve `_passengerName`:
   ```ts
   withStatus(status: SeatStatus): Seat {
     return new Seat(this._number, status, this._cabin, this._passengerName);
   }
   ```

WHY: Storing the trimmed name once at construction makes the getter pure and avoids re-trimming on every render. WhitespaceOnly normalization in `create` keeps the getter branch-free.

AVOID: Do NOT split on whitespace and use last word's last char — the spec is "first char + last char of full name" (so 'MADONNA' -> 'MA', not 'MA' from 'M' + last word). Single string lookup matches the cloud_2 behavior.

**verify:** `npm test -- Seat.test.ts` — all 5 new tests in `describe('passengerInitials')` pass, all prior Seat tests still pass.

**done:** All Seat.test.ts cases GREEN, no other test files affected.

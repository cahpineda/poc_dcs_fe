Validate and fix visual + functional parity of the seat_plan React migration against the cloud2 PHP monolith. Every visible difference, behavioral difference, and API mismatch must be cataloged in AUDIT.md, fixed in code, and re-verified before ACA-2953 can be signed off.

Phase 16 closes the loop — proving that what we built behaves identically to cloud2 from a user's seat. Output: A populated AUDIT.md, parity fixes across components/styles/mock data, and a final validation pass with 244+ passing tests, zero TypeScript errors, and zero console warnings during a full manual walkthrough.

<objective>
Audit the current React POC against the two cloud2 reference screenshots (`docs/screenshots/seat_plan.png` and `seat_plan2.png`) and fix every visual/layout discrepancy. Phase 17 is visual-only — no new business features, no new mock data, no new endpoints. Every change is in `src/presentation/components/*` or `src/styles/*`.

Specific gaps already identified from screenshot analysis:
1. **Seat input field** — small input above the seat map letting agents type a seat number to jump/select; not present in the React POC.
2. **Print link** — "Print Seat Plan (Graphical Printers Only)" link rendered above the map; not present.
3. **Row numbers on both sides** — screenshots show row labels on left AND right of each row; current `SeatRow.tsx` renders left only.
4. **Reseat mode banner styling** — current banner shows passenger name as plain text; cloud2 shows passenger name + "NOT CHECKEDIN" yellow badge styled like the passenger-list status pill.
5. **Active passenger highlight** — when a passenger is selected/being reseated, their current seat shows a distinctive yellow highlight/pulse in cloud2; ours does not.
6. **Pixel-level color check** — every seat cell color for every status must be verified against the screenshots with DevTools color picker; document any drift.

Out of scope (deferred to Phase 18): the left passenger list panel and SEAT FILTERS section. These are functional features, not visual fixes.

Output: a populated SCREENSHOT-AUDIT.md, the four visible additions (input, print link, dual row numbers, banner badge, active highlight), and zero regressions in tests/types/console.
</objective>

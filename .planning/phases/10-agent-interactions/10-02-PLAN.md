Add hooks alongside `useSeatAssign`:

Replace the stub callbacks (added in plan 10-01) with real implementations:
```ts
function handleBlock(seat: Seat) {
  blockMutation.mutate(
    BlockSeatCommand.create({ seatNumber: String(seat.number), flightId }),
    { onSuccess: () => setSelectedSeatObj(null) }
  );
}

function handleUnblock(seat: Seat) {
  unblockMutation.mutate(
    UnblockSeatCommand.create({ seatNumber: String(seat.number), flightId }),
    { onSuccess: () => setSelectedSeatObj(null) }
  );
}
```

Pass pending flags to the drawer:
```tsx
<PassengerDetailDrawer
  seat={selectedSeatObj}
  onClose={handleDrawerClose}
  onReseat={handleReseat}
  onBlock={handleBlock}
  onUnblock={handleUnblock}
  blockPending={blockMutation.isPending}
  unblockPending={unblockMutation.isPending}
/>
```

WHY `String(seat.number)`: command VOs require `seatNumber: string`. Coerce defensively at the boundary.
WHY mutation-local `onSuccess` (instead of useEffect on isSuccess): React 18 strict mode double-fires effects; mutation callbacks fire exactly once per mutate() call.

Avoid: optimistic UI updates — the underlying hook already invalidates the seatPlan query on success. Adding optimistic state would introduce drift if the API rejects.
